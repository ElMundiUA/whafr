## Two systemic ingest issues confirmed

1. **Titles-only pattern keeps surfacing.** First seen on OWASP cheat-sheets
   (already fixed). Wave-6 found it again on **pgPedia** — encyclopedia
   entries are indexed by name but article bodies are missing. Probably the
   same root cause: the connector hit a Pydantic-validation crash on the
   first content page of the site (now caught per the ingest-fix commit),
   so we only kept the homepage / index pages. Should re-resolve after
   the role-yaml re-run (Phase 3) completes.

2. **No minimum-similarity floor on search.** Spotted by the blockchain
   agent: with the corpus catastrophically thin on a topic, the retriever
   happily returns the top-k closest cosine matches no matter how
   irrelevant — "Eric Evans / Domain Language" came back for "EIP-712",
   "SAFe" framework for "Gnosis Safe", "Risc/risk management" for
   "RISC Zero / RISC-V". This is a **product UX hazard**: confidently
   wrong > silent failure. Add a `reranker_min_score` threshold in
   `KnowledgeGraph.search` or surface an empty-result signal when all
   cross-encoder scores are below a confidence floor.
